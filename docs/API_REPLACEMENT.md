# Замена локального API на реальный

1. **Настройте окружение**
   - В `.env` задайте `VITE_API_BASE_URL=https://real-api.example.com` и, при необходимости, `VITE_API_PREFIX`.
   - Убедитесь, что `src/shared/config` подтянул обновлённые переменные (перезапустите dev-сервер).

2. **Обновите эндпоинты и контракты**
   - Измените пути в `src/shared/api/endpoints.js`, описания DTO — в `src/shared/api/types.js`.
   - При несовпадении форматов добавьте адаптеры в `src/entities/<entity>/api.js`, чтобы UI получал привычную структуру.

3. **Переключите сервисы**
   - Функции в `src/entities/account/api.js` и `src/entities/search/api.js` обращаются к клиенту из `src/shared/api/http.js`.
   - Подмените реализации или расширьте параметры, сохраняя прежний контракт для компонентов.

4. **Выключите моки**
   - Если запускался локальный API (`api/src/server.js`), остановите его или замените генераторы данных.
   - UI-моки в `src/shared/mocks` держите для сторибуков/тестов, не смешивая их с продовым клиентом.

5. **Проверьте маршруты и авторизацию**
   - Гард `RequireAuth` ожидает валидный токен и истечение `expireMs`; при другой схеме добавьте адаптер в auth-фичу.
